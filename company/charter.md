# Company Charter

## Purpose

`ai-native-studio` exists to help a solo founder build useful software through disciplined AI-native product development. The system should improve leverage without removing human accountability.

## First Pilot

The first pilot product is a permissioned personal email agent for Gmail. It will begin in shadow mode and recommend actions without changing Gmail.

## Role Model

The company has four roles:

1. Founder: human final decision-maker.
2. Product-Customer Agent: product definition and customer value.
3. Builder Agent: technical design and implementation.
4. Verifier Agent: independent evaluation and release recommendation.

No fake executive, HR, scrum-master, or management roles are part of this operating model.

## Principles

- Human approval controls scope, sensitive permissions, and releases.
- Product work is captured in version-controlled artifacts.
- Deterministic code is preferred where it is sufficient.
- Model outputs must be structured and validated where they affect behavior.
- Private data, credentials, and OAuth tokens are never committed.
- Safety boundaries are product requirements, not optional implementation details.
